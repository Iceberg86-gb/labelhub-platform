# LabelHub Demo Script

## 0:00-1:00 Project Positioning

Open `README.md` and the architecture baseline. State that LabelHub is a personal defense project built around supervision-signal governance, not a full enterprise platform.

## 1:00-3:00 Owner Flow

Create a task, import dataset items, open the schema designer, publish schema version v1, and publish the task. Show first-come first-served quota and task transition audit.

## 3:00-5:00 Labeler Flow

Open task marketplace, claim one item, answer through the renderer, trigger field-level AI assistance, let draft autosave, then submit.

## 5:00-7:00 AI + Reviewer Flow

Show the AI pre-review result with prompt trace, scores, verdict, and token metadata. Then perform initial review and review-level approval, including reject reason if demonstrating the return loop.

## 7:00-9:00 Four Differentiators

Demonstrate schema v1/v2 rendering, verdict re-derivation from Quality Ledger, export snapshot hash comparison, and AI provenance export modes.

## 9:00-10:00 Submission Package

Open OpenAPI docs, ADR list, screenshots, and deployment notes. Close by mapping the demo to the course requirements.
